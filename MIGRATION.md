# Banking Migration Guide

Step-by-step guide for migrating the Acme Banking application from a traditional
layered architecture to Hexagonal Architecture with HexaGlue.

## Phase 1: Discovery

Run HexaGlue's audit plugin on the legacy codebase to get an initial assessment.
The audit identifies anti-patterns: anemic domain models, JPA annotations in business
logic, missing port boundaries, and primitive obsession.

## Phase 2: Hexagonal Restructuration

Reorganize the codebase into hexagonal boundaries:

- **banking-core**: Pure domain model (zero framework dependencies)
- **banking-persistence**: JPA infrastructure (generated by HexaGlue)
- **banking-service**: Application services implementing use cases
- **banking-api**: REST controllers (generated by HexaGlue)
- **banking-app**: Spring Boot assembly module

Key decisions:
- Domain module depends only on `jmolecules-hexagonal-architecture` for annotations
- Ports defined as interfaces in `banking-core` (in/out)
- Application services in a dedicated module, not mixed with domain

## Phase 3: Domain Purification

Transform the anemic domain model into a rich one:

- Extract Value Objects: `Money`, `Email`, `Iban`, `Address`
- Introduce Typed IDs: `AccountId`, `CustomerId`, `TransactionId`, etc.
- Add factory methods and business logic to domain entities
- Remove all JPA annotations from domain classes
- Add `reconstitute()` methods for persistence hydration

## Phase 4: Code Generation

Configure HexaGlue plugins to generate infrastructure:

- **JPA Plugin**: entities, repositories, mappers, adapters → `banking-persistence`
- **REST Plugin**: controllers, DTOs, exception handler → `banking-api`
- **Audit Plugin**: architecture compliance report
- **Living Doc Plugin**: architecture documentation

Configuration: see `hexagonal/hexaglue.yaml`.

## Key Architectural Rules

1. Domain classes have zero framework dependencies
2. Ports define the boundary between domain and infrastructure
3. Infrastructure adapters are generated, not hand-written
4. Application services orchestrate domain operations through ports
5. Multi-module reactor ensures cross-module consistency
