# Acme Banking - Case Study

Enterprise banking application used as a case study for progressive migration
to Hexagonal Architecture with [HexaGlue](https://hexaglue.io).

## Structure

```
├── legacy/       # Original monolithic application (technical layers)
├── hexagonal/    # Migrated application (hexagonal architecture + HexaGlue)
```

## Quick Start

### Prerequisites

- **JDK 21** required

### Legacy Application

```bash
cd legacy
JAVA_HOME="/opt/homebrew/opt/openjdk@21/libexec/openjdk.jdk/Contents/Home" mvn clean compile
```

### Hexagonal Application

```bash
cd hexagonal
JAVA_HOME="/opt/homebrew/opt/openjdk@21/libexec/openjdk.jdk/Contents/Home" mvn clean verify
```

Run the application:

```bash
cd hexagonal
JAVA_HOME="/opt/homebrew/opt/openjdk@21/libexec/openjdk.jdk/Contents/Home" ./mvnw spring-boot:run -pl banking-app
```

Then open [http://localhost:8080/swagger-ui.html](http://localhost:8080/swagger-ui.html).

## Migration Summary

| Aspect | Legacy | Hexagonal |
|--------|--------|-----------|
| Architecture | Technical layers | Hexagonal (ports & adapters) |
| Domain model | Anemic (JPA entities) | Rich domain model |
| Value objects | Primitives | Typed IDs, Money, Email, Iban |
| Infrastructure | Mixed with domain | Generated by HexaGlue |
| Modules | 5 (by technical layer) | 5 (by architectural role) |

See [MIGRATION.md](MIGRATION.md) for a detailed migration guide.
